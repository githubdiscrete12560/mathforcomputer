Quadrant III จตุภาคที่ III

Both x and y are negative (-, -) ทั้ง x และ y เป็นลบ (-, -)

Quadrant IV จตุภาคที่ IV

x is positive, y is negative (+, -) x เป็นบวก, y เป็นลบ (+, -)

Step-by-Step Plotting Instructions for Point (3, -2) คำแนะนำการลงจุดทีละขั้นตอนสำหรับจุด (3, -2)

  1. Start at the origin (0, 0) เริ่มต้นที่จุดกำเนิด (0, 0)
  2. Move 3 units to the right (positive x-direction) เลื่อน 3 หน่วยไปทางขวา (ทิศทาง x บวก)
  3. Move 2 units down (negative y-direction) เลื่อน 2 หน่วยลงไป (ทิศทาง y ลบ)
  4. Mark the point - it's in Quadrant IV ทำเครื่องหมายจุด - อยู่ในจตุภาคที่ IV

Cartesian Coordinate System ระบบพิกัดคาร์ทีเซียน

Historical Context บริบททางประวัติศาสตร์

Named after French mathematician René Descartes (1596-1650), who revolutionized mathematics by connecting algebra and geometry through coordinate systems. ตั้งชื่อตามนักคณิตศาสตร์ชาวฝรั่งเศส เรอเน เดการ์ต (1596-1650) ผู้ที่ปฏิวัติคณิตศาสตร์โดยเชื่อมโยงพีชคณิตและเรขาคณิตผ่านระบบพิกัด

📏 X-Axis Properties 📏 คุณสมบัติแกน X

Horizontal axis (abscissa)
• Positive direction: right
• Negative direction: left
• Represents independent variable แกนแนวนอน (อับซิสซา)
• ทิศทางบวก: ขวา
• ทิศทางลบ: ซ้าย
• แสดงตัวแปรอิสระ

📐 Y-Axis Properties 📐 คุณสมบัติแกน Y

Vertical axis (ordinate)
• Positive direction: up
• Negative direction: down
• Represents dependent variable แกนแนวตั้ง (ออร์ดิเนต)
• ทิศทางบวก: ขึ้น
• ทิศทางลบ: ลง
• แสดงตัวแปรตาม

🎯 Origin Significance 🎯 ความสำคัญของจุดกำเนิด

Point (0,0)
• Intersection of both axes
• Reference point for all coordinates
• Zero point for measurements จุด (0,0)
• จุดตัดของแกนทั้งสอง
• จุดอ้างอิงสำหรับพิกัดทั้งหมด
• จุดศูนย์สำหรับการวัด

⚙️ System Properties ⚙️ คุณสมบัติของระบบ

Key Features:
• Perpendicular axes
• Uniform scale units
• Infinite extent in all directions
• Unique point representation คุณลักษณะสำคัญ:
• แกนตั้งฉาก
• หน่วยมาตราส่วนสม่ำเสมอ
• ขยายไม่สิ้นสุดในทุกทิศทาง
• การแสดงจุดที่ไม่ซ้ำกัน

5-Step Plotting Instructions for Any Point (a, b) คำแนะนำ 5 ขั้นตอนสำหรับการลงจุดใดๆ (a, b)

  1. Locate the origin (0, 0) where the axes intersect หาจุดกำเนิด (0, 0) ที่แกนทั้งสองตัดกัน
  2. From origin, move |a| units along x-axis (right if a > 0, left if a < 0) จากจุดกำเนิด เลื่อน |a| หน่วยตามแกน x (ขวาถ้า a > 0, ซ้ายถ้า a < 0)
  3. From that position, move |b| units parallel to y-axis (up if b > 0, down if b < 0) จากตำแหน่งนั้น เลื่อน |b| หน่วยขนานกับแกน y (ขึ้นถ้า b > 0, ลงถ้า b < 0)
  4. Mark the final position with a point ทำเครื่องหมายตำแหน่งสุดท้ายด้วยจุด
  5. Label the point as (a, b) ติดป้ายจุดเป็น (a, b)

Distance Between Points ระยะทางระหว่างจุด

Distance Formula: d = √[(x₂-x₁)² + (y₂-y₁)²] สูตรระยะทาง: d = √[(x₂-x₁)² + (y₂-y₁)²]

🧮 Mathematical Foundation 🧮 รากฐานทางคณิตศาสตร์

The distance formula is derived from the Pythagorean theorem. When we connect two points, we form a right triangle where the distance is the hypotenuse, and the differences in x and y coordinates are the legs. สูตรระยะทางมาจากทฤษฎีบทพีทาโกรัส เมื่อเราเชื่อมจุดสองจุด เราจะได้สามเหลี่ยมมุมฉาก โดยระยะทางคือด้านตรงข้ามมุมฉาก และความแตกต่างของพิกัด x และ y คือด้านประกอบมุมฉาก

Complete Worked Example: Distance from A(1,2) to B(4,6) ตัวอย่างที่ทำสมบูรณ์: ระยะทางจาก A(1,2) ถึง B(4,6)

  1. Identify coordinates: A(1,2) means x₁=1, y₁=2 and B(4,6) means x₂=4, y₂=6 ระบุพิกัด: A(1,2) หมายถึง x₁=1, y₁=2 และ B(4,6) หมายถึง x₂=4, y₂=6
  2. Calculate differences: x₂-x₁ = 4-1 = 3 and y₂-y₁ = 6-2 = 4 คำนวณผลต่าง: x₂-x₁ = 4-1 = 3 และ y₂-y₁ = 6-2 = 4
  3. Square the differences: (3)² = 9 and (4)² = 16 ยกกำลังสองของผลต่าง: (3)² = 9 และ (4)² = 16
  4. Add the squares: 9 + 16 = 25 บวกค่ากำลังสอง: 9 + 16 = 25
  5. Take square root: √25 = 5 units หารากที่สอง: √25 = 5 หน่วย

Practice Problem: Find distance from C(-2,1) to D(3,-2) ปัญหาฝึกหัด: หาระยะทางจาก C(-2,1) ถึง D(3,-2)

Solution: วิธีแก้:

d = √[(3-(-2))² + (-2-1)²] = √[(5)² + (-3)²] = √[25 + 9] = √34 ≈ 5.83 units d = √[(3-(-2))² + (-2-1)²] = √[(5)² + (-3)²] = √[25 + 9] = √34 ≈ 5.83 หน่วย

Function Theory ทฤษฎีฟังก์ชัน

A function f maps every element in the domain to exactly one element in the range ฟังก์ชัน f จับคู่ทุกสมาชิกในโดเมนไปยังสมาชิกเพียงหนึ่งตัวในเรนจ์

🎯 Domain 🎯 โดเมน

Definition: The set of all possible input values (x-values) for which the function is defined.

Example: For f(x) = √x, domain is x ≥ 0
Finding Domain: Look for restrictions like division by zero, square roots of negatives นิยาม: เซตของค่าอินพุตที่เป็นไปได้ทั้งหมด (ค่า x) ที่ฟังก์ชันกำหนดไว้

ตัวอย่าง: สำหรับ f(x) = √x, โดเมนคือ x ≥ 0
การหาโดเมน: มองหาข้อจำกัดเช่น การหารด้วยศูนย์, รากที่สองของจำนวนลบ

🎪 Range 🎪 เรนจ์

Definition: The set of all possible output values (y-values) that the function can produce.

Example: For f(x) = x², range is y ≥ 0
Finding Range: Analyze function behavior, find minimum/maximum values นิยาม: เซตของค่าเอาต์พุตที่เป็นไปได้ทั้งหมด (ค่า y) ที่ฟังก์ชันสามารถผลิตได้

ตัวอย่าง: สำหรับ f(x) = x², เรนจ์คือ y ≥ 0
การหาเรนจ์: วิเคราะห์พฤติกรรมฟังก์ชัน หาค่าต่ำสุด/สูงสุด

Function vs. Relation: Key Differences ฟังก์ชัน vs. ความสัมพันธ์: ความแตกต่างหลัก

✅ Function ✅ ฟังก์ชัน

• Each input (x) has exactly ONE output (y)
• Passes the vertical line test
• Example: {(1,2), (2,3), (3,4)}
• Written as y = f(x) • แต่ละอินพุต (x) มีเอาต์พุต (y) เพียงหนึ่งเดียว
• ผ่านการทดสอบเส้นตั้ง
• ตัวอย่าง: {(1,2), (2,3), (3,4)}
• เขียนเป็น y = f(x)

❌ Relation (Not a Function) ❌ ความสัมพันธ์ (ไม่ใช่ฟังก์ชัน)

• One input can have multiple outputs
• Fails the vertical line test
• Example: {(1,2), (1,3), (2,4)}
• General mapping between sets • อินพุตหนึ่งตัวสามารถมีเอาต์พุตหลายตัว
• ไม่ผ่านการทดสอบเส้นตั้ง
• ตัวอย่าง: {(1,2), (1,3), (2,4)}
• การจับคู่ทั่วไประหว่างเซต

🔄 Special Types of Functions 🔄 ประเภทพิเศษของฟังก์ชัน

🎯 One-to-One Function (Injective) 🎯 ฟังก์ชันหนึ่งต่อหนึ่ง (อินเจกทีฟ)

Definition: Each output value corresponds to at most one input value
Test: Horizontal line test - no horizontal line intersects the graph more than once
Formally: If f(a) = f(b), then a = b
Example: f(x) = 2x + 3 นิยาม: แต่ละค่าเอาต์พุตสอดคล้องกับค่าอินพุตอย่างมากหนึ่งค่า
การทดสอบ: การทดสอบเส้นนอน - ไม่มีเส้นนอนตัดกราฟมากกว่าหนึ่งครั้ง
อย่างเป็นทางการ: ถ้า f(a) = f(b) แล้ว a = b
ตัวอย่าง: f(x) = 2x + 3

🌐 Onto Function (Surjective) 🌐 ฟังก์ชันครอบคลุม (เซอร์เจกทีฟ)

Definition: Every element in the codomain is mapped to by at least one element in the domain
Test: Range equals codomain
Formally: For every y in codomain, there exists x in domain such that f(x) = y
Example: f: ℝ → ℝ, f(x) = x³ นิยาม: ทุกสมาชิกในโคโดเมนถูกจับคู่โดยสมาชิกอย่างน้อยหนึ่งตัวในโดเมน
การทดสอบ: เรนจ์เท่ากับโคโดเมน
อย่างเป็นทางการ: สำหรับทุก y ในโคโดเมน จะมี x ในโดเมนที่ f(x) = y
ตัวอย่าง: f: ℝ → ℝ, f(x) = x³

💎 Bijective Function 💎 ฟังก์ชันไบเจกทีฟ

Definition: Both one-to-one AND onto
Properties: Has an inverse function
Perfect correspondence: Each input maps to exactly one output, and each output comes from exactly one input
Example: f(x) = 2x (from ℝ to ℝ) นิยาม: ทั้งหนึ่งต่อหนึ่งและครอบคลุม
คุณสมบัติ: มีฟังก์ชันผกผัน
การสอดคล้องที่สมบูรณ์: แต่ละอินพุตจับคู่กับเอาต์พุตเพียงหนึ่งตัว และแต่ละเอาต์พุตมาจากอินพุตเพียงหนึ่งตัว
ตัวอย่าง: f(x) = 2x (จาก ℝ ไป ℝ)

⚠️ Neither One-to-One nor Onto ⚠️ ไม่ใช่หนึ่งต่อหนึ่งและไม่ครอบคลุม

Definition: Function that fails both tests
Characteristics: Some outputs have multiple inputs, some codomain elements are never reached
Example: f: ℝ → ℝ, f(x) = x² (fails horizontal line test, negative numbers never reached) นิยาม: ฟังก์ชันที่ไม่ผ่านการทดสอบทั้งสอง
ลักษณะ: เอาต์พุตบางตัวมีอินพุตหลายตัว สมาชิกบางตัวในโคโดเมนไม่เคยถูกเข้าถึง
ตัวอย่าง: f: ℝ → ℝ, f(x) = x² (ไม่ผ่านการทดสอบเส้นนอน ไม่สามารถเข้าถึงจำนวนลบได้)

Visual Tests for Function Properties การทดสอบด้วยภาพสำหรับคุณสมบัติของฟังก์ชัน

📊 Vertical Line Test 📊 การทดสอบเส้นตั้ง

Purpose: Determines if a relation is a function
Rule: No vertical line should intersect the graph more than once
If it passes: It's a function จุดประสงค์: กำหนดว่าความสัมพันธ์เป็นฟังก์ชันหรือไม่
กฎ: ไม่มีเส้นตั้งควรตัดกราฟมากกว่าหนึ่งครั้ง
ถ้าผ่าน: เป็นฟังก์ชัน

📈 Horizontal Line Test 📈 การทดสอบเส้นนอน

Purpose: Determines if a function is one-to-one
Rule: No horizontal line should intersect the graph more than once
If it passes: Function is injective (one-to-one) จุดประสงค์: กำหนดว่าฟังก์ชันเป็นหนึ่งต่อหนึ่งหรือไม่
กฎ: ไม่มีเส้นนอนควรตัดกราฟมากกว่าหนึ่งครั้ง
ถ้าผ่าน: ฟังก์ชันเป็นอินเจกทีฟ (หนึ่งต่อหนึ่ง)

Operations with Variables การดำเนินการกับตัวแปร

➕➖ Addition & Subtraction ➕➖ การบวกและลบ

Combine like terms: 3x + 5x = 8x รวมพจน์ที่เหมือนกัน: 3x + 5x = 8x

✖️➗ Multiplication & Division ✖️➗ การคูณและหาร

Work with coefficients: 4x ÷ 2 = 2x ทำงานกับสัมประสิทธิ์: 4x ÷ 2 = 2x

📦 Distributive Property 📦 สมบัติการแจกแจง

Expand expressions: a(b + c) = ab + ac กระจายนิพจน์: a(b + c) = ab + ac

🔗 Combining Terms 🔗 การรวมพจน์

Simplify complex expressions systematically ลดรูปนิพจน์ที่ซับซ้อนอย่างเป็นระบบ

Complete Step-by-Step Example ตัวอย่างทีละขั้นตอนที่สมบูรณ์

Simplify: 4(2x + 3) - 2(x - 1) ลดรูป: 4(2x + 3) - 2(x - 1)

  1. Apply distributive property: 4(2x) + 4(3) - 2(x) - 2(-1) ใช้สมบัติการแจกแจง: 4(2x) + 4(3) - 2(x) - 2(-1)
  2. Multiply: 8x + 12 - 2x + 2 คูณ: 8x + 12 - 2x + 2
  3. Combine like terms: (8x - 2x) + (12 + 2) รวมพจน์ที่เหมือนกัน: (8x - 2x) + (12 + 2)
  4. Final answer: 6x + 14 คำตอบสุดท้าย: 6x + 14

Solving Equations การแก้สมการ

Systematic Approach to Isolating Variables วิธีการเป็นระบบในการแยกตัวแปร

Example 1: Simple Linear Equation ตัวอย่างที่ 1: สมการเชิงเส้นง่าย

Solve: 2x + 5 = 13 แก้: 2x + 5 = 13

  1. Subtract 5 from both sides: 2x + 5 - 5 = 13 - 5 ลบ 5 จากทั้งสองข้าง: 2x + 5 - 5 = 13 - 5
  2. Simplify: 2x = 8 ลดรูป: 2x = 8
  3. Divide both sides by 2: x = 4 หารทั้งสองข้างด้วย 2: x = 4
  4. Check: 2(4) + 5 = 8 + 5 = 13 ✓ ตรวจสอบ: 2(4) + 5 = 8 + 5 = 13 ✓

Example 2: Multi-step Equation ตัวอย่างที่ 2: สมการหลายขั้นตอน

Solve: 3(x - 2) + 4 = 16 แก้: 3(x - 2) + 4 = 16

  1. Distribute: 3x - 6 + 4 = 16 แจกแจง: 3x - 6 + 4 = 16
  2. Combine like terms: 3x - 2 = 16 รวมพจน์ที่เหมือนกัน: 3x - 2 = 16
  3. Add 2 to both sides: 3x = 18 บวก 2 ทั้งสองข้าง: 3x = 18
  4. Divide by 3: x = 6 หารด้วย 3: x = 6
  5. Check: 3(6-2) + 4 = 3(4) + 4 = 16 ✓ ตรวจสอบ: 3(6-2) + 4 = 3(4) + 4 = 16 ✓
